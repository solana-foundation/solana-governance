# Install & Start (Server / Docker)

Deploy flow (single path):

1. Build the binary + Docker image locally
2. Run `setup.sh` to start the container using the image tag you built

---
## Step 1: Build locally (binary + Docker image)

SSH into the server where you will run the container.

```bash
cd ~/solana-governance/ncn

cargo build --release --bin verifier-service

# Build a tagged Docker image. Use an explicit tag (do not rely on `latest`).
# This tag must be the same one you enter as `IMAGE` when running `setup.sh`.
sudo docker build -f verifier-service/Dockerfile -t verifier-service:latest .
```

---
## Step 2: Start with `setup.sh`

Before you run `setup.sh`, generate a `METRICS_AUTH_TOKEN` (required for `/admin/stats`):

```bash
# Generates a random hex token (copy/paste the output into the prompt)
METRICS_AUTH_TOKEN="$(openssl rand -hex 32)"
echo "$METRICS_AUTH_TOKEN"
```

Run:

```bash
cd ~/solana-governance/ncn/verifier-service/src/scripts
sudo bash setup.sh
```

When prompted, enter:

- `IMAGE` = `verifier-service:latest` (the tag you built above)
- `OPERATOR_PUBKEY`
- `METRICS_AUTH_TOKEN`
- `PORT_HOST` (example: `8000`, which maps `8000 -> 3000`)

<Callout type="info">
  Your `setup.sh` starts a container named `verifier` and exposes `PORT_HOST -> 3000`, persisting data under `/srv/verifier/data`.
</Callout>

---
## Step 3: Verify

Replace `<PORT_HOST>` with your `PORT_HOST`:

```bash
sudo docker ps
curl -i http://127.0.0.1:<PORT_HOST>/healthz
curl -s http://127.0.0.1:<PORT_HOST>/version
```

If it fails, check logs:

```bash
sudo docker logs --tail=200 verifier
```
